# Cardmong 게임 설계 문서

## 게임 컨셉

카드를 수집하고, 선택한 카드가 살아있는 몬스터로 변해 자동으로 전투하는 모바일 게임.
Pokémon + Auto Battler + 카드 컬렉션 RPG 조합.

## 전체 아키텍처 요약

```
┌─────────────────────────────────────┐
│         Unity 클라이언트            │
│         (Android / iOS)             │
│                                     │
│  - 카드 선택 UI                     │
│  - 전투 애니메이션 재생             │
│  - 인벤토리 / 덱 빌딩 UI           │
└────────────────┬────────────────────┘
                 │ REST API (HTTPS)
┌────────────────▼────────────────────┐
│         Spring Boot 서버            │
│                                     │
│  - 인증 (JWT)                       │
│  - 카드 / 덱 / 인벤토리 관리       │
│  - 전투 시뮬레이션 엔진             │
│  - 랭킹 / 매치메이킹               │
└────────┬───────────────┬────────────┘
         │               │
┌────────▼──────┐ ┌──────▼────────────┐
│    MySQL      │ │  Redis            │
│  영구 데이터  │ │  랭킹 캐시 / 세션 │
└───────────────┘ └───────────────────┘
```

## 전투 흐름 요약

```
1. 클라이언트: 덱 선택 후 "전투 시작" 요청
2. 서버: 매칭 상대 결정 (PvP or AI)
3. 서버: 전투 시뮬레이션 실행 (틱 기반, 100ms 단위)
4. 서버: 전투 로그 생성 후 응답
5. 클라이언트: 로그를 받아 애니메이션 재생
```

## MVP 범위

```
포함
├── 회원가입 / 로그인
├── 기본 카드 10~20종
├── 덱 빌딩 (최대 5장)
├── AI 상대 전투
├── 전투 결과 애니메이션
├── 경험치 / 레벨업
└── 카드 강화

제외 (Phase 2 이후)
├── PvP 매치메이킹
├── 길드
├── 카드 진화
├── 장비 시스템
├── 랭킹 리그
└── 인앱 결제
```
